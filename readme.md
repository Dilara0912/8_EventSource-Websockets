Как это все работает:
1. Пользователь заходит на страницу
   ↓
2. Появляется модальное окно "Выберите псевдоним"с полем для ввода ника
   ↓
3. Пользователь вводит ник ("Анна") и нажимает "Продолжить"
   ↓
4. 🚀 Отправляется POST-запрос на /new-user с телом { "name": "Анна" }
   ↓
5. Сервер проверяет:
   - Если "Анна" уже есть в userState → возвращает ошибку
   - Если "Анна" свободна → создает пользователя с UUID и возвращает успех { status: "ok", user: {...} }
   ↓
6. В зависимости от ответа:
   - Ошибка → показываем сообщение "Это имя уже занято"
   - Успех → сохраняем данные пользователя и открываем WebSocket-соединение ws://localhost:3000
   ↓
7. Сервер присылает список текущих пользователей
   ↓
8. Чат готов к работе
   ↓
9. Пользователь пишет сообщение → отправляем через WebSocket { type: "send", ... }
   ↓
10. Сервер рассылает сообщение всем клиентам
    ↓
11. Все клиенты отображают сообщение

 Полная схема работы клиентской части:
  1. Пользователь открывает страницу
   ↓
1. new Chat(root).init()
   ↓
2. bindToDOM() — создает HTML
   ↓
3. registerEvents() — вешает обработчики
   ↓
4. showModal() — показывает модальное окно
   ↓
5. Пользователь вводит ник и нажимает "Продолжить"
   ↓
6. onEnterChatHandler() — отправляет POST-запрос
   ↓
7. Сервер возвращает данные пользователя
   ↓
8. setupWebSocket() — открывает WebSocket
   ↓
9.  hideModal() — скрывает окно
    ↓
10. Пользователь пишет сообщение → sendMessage()
    ↓
11. WebSocket отправляет сообщение на сервер
    ↓
12. Сервер рассылает всем клиентам
    ↓
13. handleWebSocketMessage() → renderMessage()
    ↓
14. Сообщение появляется в чате
